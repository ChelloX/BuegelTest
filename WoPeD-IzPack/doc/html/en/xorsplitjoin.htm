XOR-split-join transitions

An XOR-split-join transition is an operator transition introduced by the class of workflow nets. It has two or more in- and output places. It is the ending and starting point of an alternative execution. According to the occurrence rule of Petri nets, the following behaviour is defined:

  • if there is at least one token on one input place, the number of tokens on this place is decreased by one and the token is added to the selected output place.

Creating

To create a new XOR-split-join transition in an open editor use one of the following commands from the edit menu:

  • Left-click on the associated symbol  and then to the position where the transition shall appear
    or
  • Right-click at the position where the transition shall appear and select " XOR-split-join" from the context menu

After this, you can fill-out the small editing box to assign a name to the new transition or leave this to the automatic naming mechanism of WoPeD.
