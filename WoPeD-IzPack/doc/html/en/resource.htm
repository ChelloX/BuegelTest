Resources

Besides the process model, the resource model is another important description view of workflow management systems. Resources can be humans, machines, raw material or other objects which are not always available (e. g. humans usually can perform only one task at a time) and/or must be utilized exclusively (printer, fax, ...).

The individual resources are called resource objects. Resources are usually organized in resource classes which can be defined as roles or groups. WoPeD supports resource modelling in the following way:

  • Resource classes and objects can be defined in the resource editor
  • Resource classes can be assigned to transitions of the Petri net by a resource mapping mechanism
