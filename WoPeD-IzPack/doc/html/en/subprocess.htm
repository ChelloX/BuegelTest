Subprocesses

This feature allows the modularization of large Petri net models. A double-border transition symbol    is used to refer to another Petri net which is to be inserted instead of this placeholder.

For transitions, the following behaviour is defined:

  • if there is at least one token on the input place, one token is removed from this place and one token is added to the output place.

Creating

To create a new subprocess in an open editor use one of the following commands:

  • Left-click on the associated toolbar symbol  and then to the position where the subprocess shall appear
    or
  • Right-click at the position where the subprocess shall appear and choose the command " Subprocess" from the context menu

Opening

To open a subprocess in an open editor use one of the following commands:

  • Right-click at the subprocess and choose the command " Open Subprocess" from the context menu

The subprocess editor

A subprocess editor is represented by a modal window providing the basic editing functions for the subprocess. The resources from the petri net can be used in a subprocess. On closing the subprocess editor the subprocess petri net is checked by the woped intern analysis and errors are reported.

The input and output from the subprocess transition are represented as read-only elements.


For more information go to the petrinet editor.
