XOR-join transitions

An XOR-join transition is an operator transition introduced by the class of workflow nets. It has two or more input places and one output place. It usually is the closing point of an alternative execution. According to the special occurrence rule for workflow nets (which differs here from the traditional occurrence rule for Petri nets), the following behaviour is defined:

  • if there is at least one token on at least one input place, one token is removed from this place and one token is added to the output place

 

Creating

To create a new XOR-join transition in an open editor use one of the following commands from the edit menu:

  • Left-click on the associated symbol  and then to the position where the transition shall appear
    or
  • Right-click at the position where the transition shall appear and select " XOR-join" from the context menu

After this, you can fill-out the small editing box to assign a name to the new transition or leave this to the automatic naming mechanism of WoPeD.
