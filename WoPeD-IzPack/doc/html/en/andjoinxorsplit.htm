AND-join-XOR-split transitions

An AND-join-XOR-split transition is an operator transition introduced by the class of workflow nets. It has two or more input places and two or more output places. It is the ending point of a parallel execution and the starting point of an alternative execution. According to the occurrence rule of Petri nets, the following behaviour is defined:

  • if there is at least one token on every input place, the number of tokens on the input places is decreased by one and one token is added to the selected output place

Creating

To create a new AND-join-XOR-split transition in an open editor use one of the following commands from the edit menu:

  • Left-click on the associated symbol  and then to the position where the transition shall appear
    or
  • Right-click at the position where the transition shall appear and select " AND-join XOR-split" from the context menu

After this, you can fill-out the small editing box to assign a name to the new transition or leave this to the automatic naming mechanism of WoPeD.
