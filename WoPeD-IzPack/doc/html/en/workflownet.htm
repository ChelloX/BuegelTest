Workflow nets

Workflow Nets (WF-Nets) are a notational basis for designing business processes. They are an extension of classical Petri Nets, being established by Wil van der Aalst. A WF-Net is supposed to have a very regular structure: It must contain exactly one place without input arcs (called input place) and exactly one place without output arcs (called output place). Moreover, the net graph (extended by one "virtual" transition with an arc from the output place and an arc to the input place) must be strongly connected, i. e. from each node there must exist a directed path to every other node.

Operator transitions

Four operator transition types are added to express branching decisions in a more compact way: AND-split, AND-join, XOR-split and XOR-join, each of them being associated with a graphical symbol. Conceptionally, operator transitions are no more than shortcuts ("syntactic sugar") for an underlying equivalent standard Petri Net construct, such that the formal nature and executability is preserved.

Subprocesses

A subprocess concept is added to handle the graphical and logical complexity of large workflow process definitions by modularity. This allows workflow processes contain other WF-Nets as a subprocess. The net notation is simply extended by a subprocess transition symbol standing for a particular subprocess.

Triggers

Triggers are added to the standard Petri net notation in order to represent different kinds of interactive dependency between the process and its operative environment. Three types of trigger are distinguished (plus a fourth, "empty" trigger type). WF-nets allow to assign exactly one trigger type to every transition (including operator transitions). The trigger type is graphically represented by a small, self-explaining icon near the associated transition symbol.

Soundness

A very important property of a WF-net is soundness.

Resources

Several notations are added to map workflow tasks onto resources by assigning one or more resource classes to all resource-triggered transitions. The classes must be defined and maintained in a separate resource model, consisting of organisational hierarchies, groups and roles.

Performance Parameters

The following quantitative performance parameters  are assigned to the elements of a Workflow Net: Average number of cases per time unit (arrival rate), average execution duration of a each workflow task, branching probabilities of every arc following an XOR-split transition.
