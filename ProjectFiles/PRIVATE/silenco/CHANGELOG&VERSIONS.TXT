CHANGELOG:

-> MODEL:
	- JGraph Modelelemente und PWT Modelelemente verschmolzen

	- copy() & paste() Methode im PWTEditor implementiert (OHNE KANTEN)
	- undo() & redo() NUR im View angeschaltet

	- Editierlogik implementiert 
	- Default Name entspricht der ID des Elementes (Durchnummerierung)
	- Automatisches Routing von Kanten (Rechtsklick auf Kante)
-> GUI:
	- Taskleiste unten (Achtung: hierfür Quellcode von SCroll verändert)
	- Erweiterung in der Buttonleiste
	- Recent Files Untermenü im "File" Menü 
	  (Standardmäßig die letzen 10 Dateien)
-> SubProcessing:
	- weitere Vorbereitungen für Subprozesse getroffen (Model/Import/Export)	
-> Import/Export:
	- im Bereich der Transitionen komplett überarbeitet
	- Subprozess-Elemente sind nun einlesbar & speicherbar (inkl. Trigger)
-> Konfiguration:
	- "PWTConfguration" -Klasse zum Abspeichern von Konfigurationen 
	- Package "de.kybeidos.pwt.config" erstellt 
	- XSD "configuration.xsd" erstellt & JavaBeans generiert
	- Automatisches Einlesen und Abspeichern der Konfiguration beim Starten
	  und Beenden von PWFTool
	- Folgende Felder sind bereits implementiert:
	  Fenstergröße/Fensterposition/Homedirectory/RecentFiles/Export 
	  bzw. Import WF-Petrinetz Notation/Woflan/Edit-Name-On-Elementcreation
-> Actions:
	- Package "de.kybeidos.pwt.actions" erstellt, worin alle Actions 
	  (GUI-unabhängig) gespeichert werden
-> ... & weitere kleinere Änderungen und Bugfixe.
