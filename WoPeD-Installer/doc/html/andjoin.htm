AND-join Transitions

An AND-join transition is a special type of transition introduced by the class of Workflow Nets. It has two or more input places and one output place. It usually is the closing point of a parallel execution. According to the occurrence rule of Petri nets, the following behaviour is defined:

  • if there is at least one token on every input place, one token is removed from each of these places and one token is added to the output place

 

 

Creating

To create a new AND-join transition in an open Editor use one of the following commands:

  • Left-click on the associated toolbar symbol   and then to the position where the transition shall appear
    or
  • Right-click at the position where the transition shall appear and choose the command "Add -> AND-join" from the context menu

After this, you can fill-out the small editing box to assign a name to the new transition or leave this to the automatic naming mechanism of WoPeD.
