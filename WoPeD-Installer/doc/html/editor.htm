Editor Window

An editor is represented by one window providing the basic editing functions for Petri net models in WoPeD.

An editor can contain the following standard Petri net elements:

  • Places    are passive elements of the Petri net
  • Transitions    are active elements of the Petri net
  • Arcs    are connecting two different types of Petri net elements
  • Tokens    are data objects assigned to places

Places and transitions can have names annoted to their symbol.

 

Additionally, an editor can contain the following special workflow net elements:

  • AND-split Transitions    are starting points of a parallel branching
  • XOR-split Transitions    are starting points of an alternative branching
  • AND-join Transitions    are closing points of a parallel branching
  • XOR-join Transitions    are closing points of an alternative branching
  • Subprocess Transitions    are placeholders for subprocess definitions
  • Triggers are assigned to transitions depending on special conditions:

- Resource triggers    define that an available Resource is needed

- Message triggers    define that an external message must - arrive

- Time triggers    define that a certain time must be reached

- Transitions without trigger symbol can be executed unrestrictedly
