Token Game

The token game is the graphical representation of the basic Petri net execution rule, describing the following dependency between transitions, places and tokens:

  • A transition is enabled, if all of its input places have at least one token
  • An enabled transition can occur - it then removes one token from each input place and adds one token on each output place

Note that operator transitions, in particular XOR-split transitions and XOR-join transitions used in workflow nets have a different behaviour with respect to this occurrence rule.

Starting and stopping the Token Game

In order to start the token game, press the   button on the toolbar. The tokengame remotecontrol is launched for the net in the active editor window. When clicking play in the remotecontrol the net graph becomes read-only (represented by a beige window background) and all transitions which are enabled under the current marking are highlighted with a small red inscription. To stop the token game, just close the remotecontrol or press the   button again, the editor window background returns to its normal color and the net becomes editable again.

If there is initially at least one transition enabled, the user has to choose the next transition to occur by clicking on the associated control element within the net graph. The following two situations can be distinguished:

  • Choosing a normal or AND transition to occur
  • Choosing an XOR transition to occur
