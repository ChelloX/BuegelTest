Reachability Graph Extras

Coverability

Computing a reachability graph can be very difficult to handle. Some graph tend to get very big. And some graph are not really solvable in available time. There is special case of graph that are not ending while computing them. It has to do with unboundedness of a petri net. A unbounded petri net has places, that can get infinit number of tokens.
To handle this situation and to nevertheless get the graph drawn you can compute a coverability graph. This graph is just like a normal reachability graph but it can handle unbounded places. It represents unbounded places by a "w" in markings, where a place gets unbound.
