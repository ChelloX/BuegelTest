Transition property dialog

Opening this dialog makes it possible to specify properties of a transition in the editor.

 

General properties:

The following general properties can be specified:

  • Name of the transition - an arbitrary string
  • Id# of the transition (not editable) - unique, system-assigned identifier
  • Branching type (  not yet implemented)
  • Execution time and time unit of associated task (   not yet implemented)
  • Trigger type - resource, message, time or no trigger
  • Resource mapping - assign a role and one group to a resource-triggered transition (will be disabled for other trigger types). Whenever this is done, the number of associated resource objects is shown in the dialog. The definition of appropriate resource classes and the assignment of resource objects to these classes can be done in the resource editor. Be sure to avoid those assigments resulting in zero.

 

 

 

BPEL properties:

One of the following BPEL-activities can be assigned to a transition:

  • assign
  • invoke
  • receive
  • reply
  • wait
