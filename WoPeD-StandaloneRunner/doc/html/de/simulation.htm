Simulation

Bei der Simulation handelt es sich um eine Methode der quantitativen Analyse, bei der das Geschäftsprozessmodel gewissermaßen ausgeführt wird, wobei begleitend Daten gesammelt werden.

Das Prozessmodell muss dafür die mittleren Bedienzeiten der Transitionen, die Verzweigungswahrscheinlichkeiten bei XOR-Splits und eine Gruppen-Rollen-Zuordnung zu den Transitionen angeben.

Nach dem Öffnen des Simulationsdialogs erscheinen eine Menge von Informationen. Sie werden in Folgenden in der Leserichtung von links nach rechts und von oben nach unten vorgestellt.

Als erstes muss die mittlere Ankunftsrate von neuen Fällen pro Periode λ und dann die Periode π selbst eingegeben werden. Im Kombinationslistenfeld sollte die richtige Zeiteinheit für die Periode ausgewählt werden. Anschließend wird die Warteschlangendisziplin ausgewählt und dann die Anzahl der Simulationsläufe eingetragen. Als nächstens wird die Abbruchbedingung spezifiziert. Es besteht die Auswahl zwischen Abbruch, wenn mindestens λ Fälle fertiggestellt wurden; Abbruch, wenn die Zeit für die Periode abgelaufen ist oder Abbruch, sobald die erste der beiden Regeln zutrifft. Das geschieht durch Auswahl beider Kontrollkästchen.

Die beiden letzten Einstellungen betreffen die Wahrscheinlichkeitsverteilungen des Ankunftsprozesses und der Bedienzeiten der Server. Zur Auswahl steht eine konstante Verteilung mit Parameter ς über dem Intervall [(1-ς) λ, (1+ς) λ] um den Mittelwert λ bzw. der Bedienzeit μ, in welchem jeder Wert gleich wahrscheinlich ist. ς muss eine Ganzzahl zwischen 0 und 100 sein. Es kann auch eine Poissonverteilung mit Parameter λπ gewählt werden. Die dritte Alternative ist eine Normalverteilung mit Mittelwert λ und Standardabweichung σ, welches als Parameter anzugeben ist.

Wenn eine andere Standardzeiteinheit als 1 Minute gewünscht wird, kann sie im Zeitmodell-Dialog eingestellt werden. Der Dialog ist über die Schaltfläche [Zeitmodell] erreichbar.

Nachdem alle notwendigen Daten eingegeben wurden, kann die Simulation mit Klick auf [Starten] begonnen werden. Es erscheint ein Wartedialog, bis die Berechnugn abgeschlossen wurde. Dann erscheinen die Ergebnisse in den beiden Tabellen. Außerdem sind jetzt drei weitere Schaltflächen aktiviert. In den Tabellen sind sämtliche statistische Daten des Prozesses und der einzelnen Transitionen / Server aufgeführt und die berechneten Auslastungsgrade der Ressourcen.

Die Spaltenüberschriften der ersten Tabelle haben folgende Bedeutung:

  • L      die mittlere Anzahl von Fällen im Prozess bzw. am Server

  • Lq     die durchschnittliche Anzahl wartender Fälle

  • Ls     die durchschnittliche Anzahl bedienter Fälle

  • W     die mittlere Durchlaufzeit (Bedien- und Wartezeiten)

  • Wq    die mittlere Wartezeit

Wird eine der Schaltflächen in der Details-Spalte betätigt, öffnet sich ein neues Fenster mit zusätzlichen Informationen.

Nach Ablauf der Simulation kann das Protokoll eingesehen werden, indem die Schaltfläche [Protokoll] geklickt wird. Die Ergebnisse können über [Exportieren] in CSV-Dateien gespeichert werden. Außerdem kann mittels [Diagramm] eine graphische Darstellung der Belegung der einzelnen Server mit Ressourcen über den Zeitablauf eingesehen werden.

Mit [Schließen] wird der Simulationsdialog geschlossen.
