Transitions

A transition used by the class of workflow nets has one input place and one output place. Note that this is a restriction to the traditional Petri net notation, where transitions may have arbitrary many input and output places.

For transitions, the following behaviour is defined:

  • if there is at least one token on the input place, one token is removed from this place and one token is added to the output place.

  Note that any kind of branching should not be expressed by a normal transition but by one of the provided operator transitions instead.

Creating

To create a new transition in an open editor use one of the following commands:

  • Left-click on the associated toolbar symbol  and then to the position where the transition shall appear
    or
  • Right-click at the position where the transition shall appear and choose the command " Transition" from the context menu
